### 修复方法
- 首先尝试 `feishu_update_doc(mode="overwrite")` 整体重写 → **失败**，overwrite 生成的内容仍存在空条目
- 然后尝试 `feishu_update_doc(mode="replace_range")` 精确替换 → **部分成功**，部分内容仍然空
- 最终使用 `feishu_update_doc(mode="insert_after")` 在空条目后插入代码块 → **成功**
- 逐个条目修复，共 14 次 API 调用

### 教训（关键发现！）
1. **overwrite 模式的限制**：即使返回成功，overwrite 后的文档仍可能丢失部分条目内容。原因可能是 markdown → 飞书文档转换过程中某些格式被处理/丢失
2. **replace_range 的局限性**：当文档中的文本格式与传入的 selection 不完全一致（如缩进、换行差异）时，replace_range 可能匹配错误位置或不生效
3. **insert_after 是最可靠的方法**：对于"标题/冒号后缺少内容"的问题，`insert_after` 模式可以精确定位并追加内容，是最稳健的修复策略
4. **飞书文档 API 的内容差异**：API 返回的 markdown 格式与原始写入的格式可能有细微差异（如 `&` 被转义为 `\u0026`，缩进被移除等），这影响 replace_range 的匹配
